# Trackpad DJ

macOS 전용 실험적 DJ/믹싱 앱. 랩톱 트랙패드를 주 컨트롤 서피스로 사용합니다.

> 프로 DJ 하드웨어를 대체하는 것이 목표가 아닙니다.
> 트랙패드만으로 **간단한 2덱 믹싱이 가능한가**를 증명하는 HCI 실험 프로젝트입니다.

---

## 요구 사항

- macOS 13+
- Swift 5.9+ (`swift --version`)
- 내장 트랙패드 (멀티터치 간접 입력 필요)

---

## 빌드 & 실행

```bash
swift build
.build/debug/TrackpadDJ
```

## 현재 상태 — Phase 3: 메인 믹싱 UI

2덱 오디오 엔진과 트랙패드/키보드 컨트롤이 완성된 상태입니다.

**화면 레이아웃:**

```
┌─────────────────────────────────┐
│   VOL A ████░░   VOL B ░░████   │  ← 상단 스트립: 채널 볼륨 페이더
├────────────────┬────────────────┤
│  Track Name ▶  │  ▶ Track Name  │  ← 트랙명 + 재생 상태 + 남은 시간
│  ▓▓▓▓░░░░░░░│  │  │░░░▓▓▓▓▓░░  │  ← 파형 + 플레이헤드
│                │                │
├────────────────┴────────────────┤
│         A  │  A+B  │  B         │  ← 하단 스트립: 크로스페이더 (3모드)
└─────────────────────────────────┘
```

---

## 컨트롤

### 트랙패드

| 존 | 제스처 | 기능 |
|-----|--------|------|
| 상단 스트립 좌 | 1핑거 수직 | 덱 A 볼륨 |
| 상단 스트립 우 | 1핑거 수직 | 덱 B 볼륨 |
| 덱 A/B 존 | 1핑거 터치 | 오디오 프리즈 (스크래치 준비) |
| 덱 A/B 존 | 1핑거 수평 스와이프 | 스크래치 (속도/방향 비례) |
| 덱 A/B 존 | 2핑거 수직 | 로우패스 필터 |
| 하단 스트립 | 1핑거 수평 | 크로스페이더 |

### 키보드

| 키 | 기능 |
|----|------|
| Q / W | 덱 A / B 트랙 로드 (파일 선택) |
| A / S | 덱 A / B 재생/정지 |
| Z / X | 덱 A / B 큐 (시작점 복귀) |
| E / D | 덱 A 볼륨 올리기 / 내리기 |
| R / F | 덱 B 볼륨 올리기 / 내리기 |
| T / G | 덱 A 필터 열기 / 닫기 |
| Y / H | 덱 B 필터 열기 / 닫기 |
| ↑ / ↓ | 덱 A 위치 앞으로 / 뒤로 (너지) |
| I / K | 덱 B 위치 앞으로 / 뒤로 (너지) |
| ← / → | 크로스페이더 모드 전환 (A → A+B → B) |

## 프로젝트 구조

```
Sources/TrackpadDJ/
├── App/
│   ├── main.swift                    # NSApplication 진입점
│   └── AppDelegate.swift             # 윈도우 생성
├── input/
│   ├── TouchPoint.swift              # 단일 터치 포인트 (불변 구조체)
│   └── TouchSession.swift            # 활성 터치 컬렉션 (불변)
├── gestures/
│   └── Zone.swift                    # 존 정의 + 기본 레이아웃
├── audio/
│   ├── DeckProtocol.swift            # 덱 공통 인터페이스
│   ├── Deck.swift                    # AVAudioSourceNode 기반 덱
│   ├── AudioEngine.swift             # AVAudioEngine + 2덱 믹서
│   └── CrossfaderState.swift         # 크로스페이더 상태
└── ui/
    ├── TouchLabView.swift             # 메인 믹싱 뷰 (터치/키보드 + 렌더링)
    └── TouchLabViewController.swift   # 뷰 ↔ 오디오 엔진 연결
```

## 설계 원칙

- **존 기반 입력** — 트랙패드를 고정된 기능 영역으로 분할, 모호한 제스처 회피
- **불변 데이터** — TouchPoint, TouchSession 모두 불변 구조체
- **입력·오디오 분리** — 제스처 해석과 오디오 파라미터 적용을 별도 레이어로 유지
- **키보드 병행** — 모든 기능(스크래치 제외)을 키보드로도 조작 가능
