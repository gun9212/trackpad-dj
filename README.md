# Trackpad DJ

macOS 전용 실험적 DJ/믹싱 앱. 랩톱 트랙패드를 주 컨트롤 서피스로 사용합니다.

> 프로 DJ 하드웨어를 대체하는 것이 목표가 아닙니다.
> 트랙패드만으로 **간단한 2덱 믹싱이 가능한가**를 증명하는 HCI 실험 프로젝트입니다.

---

## 요구 사항

- macOS 13+
- Swift 5.9+ (`swift --version`)
- 내장 트랙패드 (멀티터치 간접 입력 필요)

---

## 빌드 & 실행

```bash
# 빌드
swift build

# 실행
swift run
# 또는
.build/debug/TrackpadDJ
```

## 현재 상태 — Phase 0: Touch Lab

트랙패드 입력을 시각화하는 디버그 화면입니다.
오디오 기능을 추가하기 전에 입력 레이어와 존 분류를 먼저 검증합니다.

**Touch Lab 화면:**

```
┌─────────────────────────────┐
│          Browse             │  ← 상단 스트립: 트랙 탐색
├──────────────┬──────────────┤
│              │              │
│   Deck A     │   Deck B     │  ← 덱 존: 조그/필터
│              │              │
├──────────────┴──────────────┤
│         Crossfader          │  ← 하단 스트립: 크로스페이더
└─────────────────────────────┘
```

트랙패드에 손가락을 올리면 해당 존에 색상 도트와 좌표가 표시됩니다.

## 프로젝트 구조

```
Sources/TrackpadDJ/
├── App/
│   ├── main.swift                 # NSApplication 진입점
│   └── AppDelegate.swift          # 윈도우 생성
├── input/
│   ├── TouchPoint.swift           # 단일 터치 포인트 (불변 구조체)
│   └── TouchSession.swift         # 활성 터치 컬렉션 (불변)
├── gestures/
│   └── Zone.swift                 # 존 정의 + 기본 레이아웃
└── ui/
    ├── TouchLabView.swift          # 존 렌더링 + 터치 시각화
    └── TouchLabViewController.swift
```

## MVP 성공 기준

- 두 트랙 로드 및 재생
- 크로스페이더로 간단한 전환
- 조그/너지 기본 동작
- 큐 포인트 트리거
- 단순 필터 1개 조작

---

## 설계 원칙

- **존 기반 입력** — 트랙패드를 고정된 기능 영역으로 분할, 모호한 제스처 회피
- **불변 데이터** — TouchPoint, TouchSession 모두 불변 구조체
- **입력·오디오 분리** — 제스처 해석과 오디오 파라미터 적용을 별도 레이어로 유지
- **단계별 진행** — Touch Lab 검증 전에 오디오 레이어 미착수
